# 知识库收口（本期：只收口 UI，编译/问答占位）

## 目标
把 `static/wiki.html` 里并存的两套知识库系统收成一套：只保留「本地知识库」（数据在用户选的本地文件夹），删除旧「Wiki 控制台」面板。编译/搜索/问答本期做成清晰的"敬请期待"占位，不假装可用。目录命名保持中文现状。

## 背景事实（已确认）
- 旧系统 `.wiki-main` 面板调用 `/api/wiki/*`，数据存应用内部 `data/wiki/`（现仅一个空 graph.json，删除零风险）。
- `setLocalSetupMode(hasActive)`（wiki.html:678）已在有库时隐藏 `.wiki-main`，所以旧面板目前只在"未创建"时露出。
- 编译/搜索/问答能力只存在于旧系统；本期不迁移，故旧后端 `/api/wiki/*` 暂保留代码（前端不再引用），不删后端，避免牵连 Agent 现有调用。

## 改动范围（本期只动前端 wiki.html）

### 1. 删除旧「Wiki 控制台」面板
- 移除 `<main class="wiki-main">`整段 DOM（来源/摘要/概念/索引/图谱 tabs、粘贴文本表单、问答框、detail 面板）。
- 移除其相关 JS：`/api/wiki/overview|pages|graph|sources|compile|search|qa` 的前端调用函数与事件绑定。
- 移除 `setLocalSetupMode` 里对 `.wiki-main` 的 toggle（连同元素一起删）。

### 2. 创建首屏 XOR 工作区（严格二选一）
- 没有任何本地知识库时：只显示 `#localCreateCard`（名称 + 保存位置 + 仅本地开关 + 创建）。
- 创建/选择后：只显示 `#localWorkspace`。
- 把现在独立的 `#localInboxCard`（投喂卡）从 onboarding 区移到工作区「收件箱」tab 内（见 3）。

### 3. 收件箱 tab = 唯一投喂入口
- 把上传文件 / 拖拽 / 导入本地路径 / 粘贴文本建来源，全部并到「收件箱」tab。
- 粘贴文本：本期落地为在 `00_收件箱/上传文件/` 写一个 `.md`（复用 `/api/local-wiki/upload` 或新增轻量写入，本期倾向复用 upload 以最小改动；若 upload 只收 multipart，则前端用 Blob 包成文件上传）。
- 保留 `/api/local-wiki/upload`、`/api/local-wiki/import-path` 现有调用。

### 4. 编译/搜索/问答占位
- 在「总览」或「收件箱」放一个 `[整理/编译]` 按钮，点击显示"敬请期待：本地编译将把收件箱整理成摘要/概念/索引（开发中）"，不调用任何接口。
- 问答入口同样占位说明，指引用户后续在 Agent 首页使用。

## 不做（本期明确排除）
- 不迁移编译/搜索/问答到本地文件夹（下一期）。
- 不动 Agent 执行器对 `data/wiki` 的读写（下一期随编译迁移一起改）。
- 不动后端 `/api/wiki/*` 路由代码（仅前端停止引用）。
- 不动顶栏/左栏导航（第三步才做）。

## 验证
1. 起服务 `python main.py`，开 `/static/wiki.html`。
2. 无库时只见创建首屏，看不到旧控制台和工作区。
3. 创建一个库 → 自动进工作区，5 个 tab 正常切换，旧控制台不再出现。
4. 收件箱 tab 内上传/导入/粘贴文本，文件确实落到所选本地文件夹的 `00_收件箱/`。
5. 点编译/问答 → 出现占位说明，无报错、无脏调用。
6. 浏览器控制台无 `/api/wiki/*`（非 local-wiki）请求、无 JS 报错。

## 风险
- wiki.html 是单文件大页，删 DOM 要同步删掉其引用的 JS 变量/函数，否则报未定义。改完用控制台无报错来兜底验证。
