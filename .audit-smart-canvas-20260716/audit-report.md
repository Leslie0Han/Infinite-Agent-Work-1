# 智能画布对抗性审查（2026-07-16）

范围：以录屏中的状态模型为基准，实际走查当前本地画布；未修改产品代码。

## 结论

当前版本已经具备对象工具栏、候选预览、九种风格、材质库、历史与场景人物等界面，但核心工作流仍不可靠。最严重的问题是：部分“选择”控件会直接提交生成；区域选择取消会丢失任务上下文；生成能力因 API 配置和输入格式问题不可完成。

## P0：会造成错误任务或不可完成

1. 视频里的 `Still` 选项一点击即提交生成，而不是只选择镜头，再由单独的“生成”确认。测试产生了失败候选，错误为输入图片无法转换为视频接口支持的格式，并暴露内部 `/static/style-previews/physical-model.png` 路径。已通过 Undo 撤销测试生成。
2. 现有 Pink Blue 候选和后续生成入口均显示“未配置 新 API 平台 的 API Key”。界面仍显示可点击的生成按钮，用户只能在点击后才发现任务无法完成。
3. 替换材质进入“选择区域”后点取消，直接关闭替换材质面板；区域、材料与描述上下文没有返回或恢复。这违反录屏中可恢复的两层选择流程。

## P1：与录屏逻辑不一致或极易误用

1. 区域选择打开的是通用“裁剪图片”编辑器（裁剪、画笔、宫格切分），不是针对当前任务的聚焦选区模式；用户难以知道最终提交的是遮罩、裁剪还是选区引用。
2. 视频功能从录屏中的明确不可执行状态，变成了可直接触发的生成面板；但没有独立提交按钮、运行状态或候选预览约束。
3. 材质库的每张卡片仅显示缩略图与无标签图标，DOM 中也没有材质名称/“作为参考”或“加入画布”的可区分语义；两个图标的作用无法预判。
4. Agent 能读到失败历史并给建议，但它的一键“创建选区”同样是直接动作，且没有完成录屏中的“几何区域选择后再确认建筑构件范围”的语义确认。
5. 项目历史仅显示单日的少数条目，卡片默认动作是“加入画布”；没有明确的预览、恢复版本和重新生成的区分。
6. 场景人物具有密度、年龄、人群特征等参数，但“身份/性别”和“人群特征”都以通用下拉呈现，缺少录屏里更清楚的任务语义与当前选择摘要。

## 已验证可用的部分

1. 选中图像会出现对象级工具栏和尺寸信息。
2. 单击候选会原位预览，未自动变成独立画布节点。
3. 生成风格面板存在录屏中的九个预设。
4. 材质库、项目历史和场景人物面板均能打开。

## 未做的破坏性验证

未双击候选“加入画布”、未上传文件、未真正调用图像生成或下载，避免改变当前画布内容或消耗外部模型资源。生成可完成性已经被现有 API Key 错误和视频输入格式错误阻断。
